## Purpose

This file tracks the major milestone sequence from planning to launch.

## Proposed Milestones

### Milestone 1. Planning Foundation Complete

Success means:

- phase-one planning docs exist
- scope is documented
- open threads are documented
- phase structure is documented

### Milestone 2. Critical Threads Closed

Success means:

- order status model is finalized
- checkout / authentication model is finalized
- email communication flow is finalized

### Milestone 3. Business Data Contract Approved

Success means:

- Excel structure is finalized
- source-of-truth rules are documented
- ownership between Excel, Sanity, and Supabase is documented

### Milestone 4. Commerce Foundation Approved

Success means:

- high-level data model is accepted
- order lifecycle is accepted
- system map is clear enough for implementation

### Milestone 5. Buyable Product Pages And Cart Working

Success means:

- PDP buyability logic works
- dual CTA logic works
- configured products can be added to cart
- items can be fully reconfigured inside cart

### Milestone 6. Checkout And Payments Working

Success means:

- checkout captures required data
- order creation works
- `Przelewy24` payment flow works end to end

### Milestone 7. Customer Panel Working

Success means:

- email OTP access works
- customer can view B2C orders
- customer can view order details and invoices
- customer can view current status and order status history
- customer can cancel eligible orders

### Milestone 8. Admin Operations Working

Success means:

- operator can view orders
- operator can update statuses
- operator can add courier / tracking info
- operator can attach invoice PDFs
- operator can manage coupons

### Milestone 9. Policy Flows Working

Success means:

- returns rules work
- cancellation rules work
- company invoice restrictions work

### Milestone 10. Launch Ready

Success means:

- key flows are tested
- critical bugs are resolved
- content/legal/payment configuration is ready
- production readiness checks are complete

## Notes

Milestones are planning checkpoints, not necessarily sprint boundaries.
